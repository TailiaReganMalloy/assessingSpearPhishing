1. PASSWORD SECURITY (db/auth.js)
   â€¢ Uses bcrypt with 10 salt rounds
   â€¢ Never stores or logs plain text passwords
   â€¢ Passwords hashed before database storage
   â€¢ Verification compares hashes, never logs them

2. DATABASE SECURITY (db/database.js)
   â€¢ All SQL queries use parameterized statements (?)
   â€¢ No string concatenation in SQL
   â€¢ User input treated as data, never code
   â€¢ Foreign key constraints for data integrity

3. SESSION SECURITY (server.js)
   â€¢ httpOnly flag prevents JavaScript access (XSS protection)
   â€¢ sameSite: strict prevents CSRF attacks
   â€¢ secure: true forces HTTPS in production
   â€¢ 24-hour expiration after inactivity

4. INPUT VALIDATION (routes/auth.js)
   â€¢ Email validation and normalization
   â€¢ Password strength enforcement (8+ characters)
   â€¢ Server-side validation (not relying on client)
   â€¢ express-validator for comprehensive checking

5. HTTP SECURITY (server.js)
   â€¢ Helmet.js sets security headers
   â€¢ Content-Security-Policy
   â€¢ X-Frame-Options (clickjacking prevention)
   â€¢ X-Content-Type-Options (MIME sniffing prevention)
   â€¢ Strict-Transport-Security

6. XSS PROTECTION (views/*.ejs)
   â€¢ EJS auto-escapes all output by default

   â€¢ No innerHTML with user input
   â€¢ Safe rendering of user-generated content
